# Sprint audit — 2026-07-19 (Fable)

Reviewer pass over the batch-3 sprint (22 commits, `origin/main..HEAD` at the time of the
audit: T16/T17/T24/T21/T4/A1/A3/T25/T14/A4, EC1-4, TL3/TL4/T22/T23/T11, the T18 and T19
agent merges, and the pass-4 evening fixes). Checklist per the standing audit brief:
correctness, serde/compat, K-031, cache-key honesty, UI wiring, docs-vs-code, tests.

## Verified clean

1. **No-panic rule** — no `unwrap`/`expect`/`panic!` outside `#[cfg(test)]` in
   `lumit-core`, `lumit-gpu`, `lumit-eval` (all sweep hits sit inside test modules).
2. **Theme discipline** — no hex/`Color32` literals outside `theme.rs`/`icons.rs`.
3. **Cache-key honesty** — `lumit-eval` hashes effect parameters generically by evaluated
   value (`EffectValue::*` walk), so every parameter this sprint added (Vignette `ramp`,
   RGB-split/spectral `tints`, Shake `motion_blur`/`mb_amount`, Datamosh
   `displacement`/`bloom`/`reset_interval`, Echo's renumbered `mode`) feeds the frame key
   with no per-effect wiring. `blend_tag` gained stable bytes 10–25 for the new blend
   modes (never reused).
4. **K-031 preview == export** — the two duplicate `BlendMode → gpu::Blend` mappers were
   collapsed to one (`shell::inspector::blend_of`; export delegates). The compositor's
   snapshot blends — previously oracle-less — now have a GPU test against a Rust reference
   (`perceptual_blend_modes_match_the_reference_formula`). The T18/T19 kernels ship with
   CPU/GPU oracles (both measured worst 1 fp16 ULP, bound ≤ 2).
5. **Snapshot blend routing** — `fs_layer_snapshot` boundaries re-checked: modes 4/5/6
   (lighten/darken/subtract) run linear; 0–3 and 7–22 run encoded via `blend_encoded`;
   no mode falls through to the wrong branch.
6. **Serde/compat** — `BlendMode` serialises by variant name (old projects load; new
   variants are additive). Every new field on UI state is session-only (not serialised).
   The echo-mode renumber and datamosh param reshape rely on the pre-release no-migration
   policy, and both are recorded (docs/02, tracker).
7. **Tests** — full workspace green at HEAD: 571 non-GPU + 55 GPU (single-threaded) = 626.
   fmt and `clippy --workspace --all-targets -D warnings` clean.
8. **`DefaultHasher` regression check** — one use (`audio_jobs_signature`) is
   session-only change detection, never persisted: safe. A guard comment was added so it
   is not copied into persisted paths (the prior audit's `cache_root_for` bug class).

## Findings

- **F1 (fixed)** — the new top-row time readout rounded the frame rate for its `ff`
  digits, drifting against true timecode at fractional rates (23.976). Swapped to the
  shared `fmt_timecode_frames`, the same formatter the Time row uses.
- **F2 (minor, open)** — a lane drag over a Retime **endpoint** key previews movement and
  then snaps back on release (the commit correctly refuses to move the structural
  [0, dur] keys). Cosmetic; exclude endpoints from the drag preview when convenient.
- **F3 (by design, eyeball)** — `snap_to_value_column` no-ops when a long label has
  already passed the row's midline; the value then packs on after the label instead of
  overlapping. Verify the mixed look reads acceptably in narrow layouts.
- **F4 (verified)** — the per-project outline-width memory rides on egui persisted
  storage; eframe storage exists and egui memory persists by default, so it survives
  restarts.
- **F5 (history note)** — the two agent worktrees branched from a stale base and both
  claimed already-taken K-numbers; reconciled on merge (Datamosh → K-164, Shake MB →
  K-165) with every code/doc reference renumbered. Their commit *subject lines* keep the
  old numbers — history only, the docs are authoritative.
- **F6 (non-issue)** — several per-frame UI paths call `store.snapshot()` (e.g. the MB
  master read in the top row). The snapshot is an Arc clone; measured cost negligible.
- **F7 (minor, eyeball)** — the Colour param row snaps its whole swatch + eyedropper +
  RGB-drag cluster to the value column; on a narrow Effect Controls panel the RGB boxes
  can clip at the right edge (they clipped before too, one column further left).
- **F8 (open follow-ups, tracked)** — TL2 column groups (owner: revisit after pass-4
  verification); EC3 for File/Layer param rows; Retime key copy/paste; removing the
  redundant accumulation-MB *effect* now the comp master exists; T14 picker for the layer
  transform's Anchor/Position; A2 fast-MB quality; A5/T2/T3/T20 owner re-verification.

## Optimisation / improvement notes (no action taken)

- The timeline rebuilds `lane_glyphs`/`prop_row_order` every frame by design (immediate
  mode); volumes are small. No hot-path allocations found beyond the established
  per-frame clones (`selected_props`), all bounded by row counts.
- `effects_rows` clones the whole effect stack per edit commit — correct (whole-stack
  `SetLayerEffects` = one undo step) and cheap at realistic stack sizes.
- The waveform strip draws one `line_segment` per bucket (≤ 2048); fine. If it ever
  shows up in profiles, batch into a single `Shape::Vec`.
